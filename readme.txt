公共字段自动填充:
    问题分析: 在多个模块中存在重复的逻辑, 如新增员工, 新增分类 ... 都会需要补充基础属性字段 (创建时间, 创建人, 更新时间, 更新人)
        这种情况导致代码冗余, 不方便后续维护
    解决方案: AOP技术--处理共性逻辑
        选择哪种通知类型? --> 前置通知, 增强mapper
        选择哪种切入点表达式? --> @annotation方式
        去粉饰新增还是更新方法? --> 判断方法注解上的属性值

    总结: aop, 自定义注解, 枚举, 反射
